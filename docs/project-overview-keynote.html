The problem

Why you can't dual-write entitlements.

Three gaps emerge the moment you give two governance planes the same data.

01 · Drift

Manual edits diverge from LF.

No continuous reconciliation, no detector, no replay. Drift accumulates silently between cycles until someone notices a missing grant.

“Who granted MODIFY on this table last Tuesday?”
02 · Identity

Same human, three ARNs.

LF speaks IAM roles, SAML federated users, and IDC groups. UC speaks SCIM emails. Mapping them is configuration, not a one-liner.

“ron.guerrero@… shows up as four different ARNs in LF”
03 · Audit

No trail of who, what, when.

Compliance can't reconstruct the chain. Field engineers can't see drift trends without a system of record for the sync itself.

“Show me every grant LF added this month”

LF stays the system of record. UC is downstream, eventually consistent, always converging.

01 · Simplicity

One moving part.

No Lambda, no EventBridge, no SQS DLQ to monitor. One Job, one schedule, one log.

02 · Correctness

Full-state diff.

Read all of LF, read all of UC, compute the delta. No class of “missed event” bug.

03 · Convergence

Drift gets corrected.

Manual UC edits on managed objects are reverted next cycle. No accumulation, no surprises.

04 · 06 · Architecture
Scope

Six surfaces, mirrored.

Five fully live in SQL today. ABAC policy creation is wired but stubbed pending the UC ABAC REST API.

MVP
01

LF-Tag dictionary

Keys and allowed values mirrored to UC tag keys, with optional namespace remap (classification → data_classification).

MVP
02

Object tags

ALTER TABLE/SCHEMA/CATALOG … SET TAGS for every LF-tagged resource. Idempotent, namespace-remapped.

MVP
03

Identity resolution

IDC groups, SAML federated users, STS assumed-role sessions, IAM roles → UC SCIM identities. Overrides for service roles.

MVP
04

Grants

SELECT, MODIFY, USE CATALOG/SCHEMA chain. Level-aware LF→UC privilege table; unsupported verbs audited.

Preview
05

ABAC policies

One UC policy per LF-Tag key. Orchestration complete; REST integration stubbed in LoggingABACPolicyAPI.

MVP
06

Audit trail

One Delta row per UC op: timestamp, op kind, principal, resource, status, latency. Drives the AI/BI Drift Dashboard.
